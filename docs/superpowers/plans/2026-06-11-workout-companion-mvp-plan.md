# Workout Companion MVP Implementation Plan

## 목표

설계서의 첫 버전 범위에 맞춰 MacBook 운동 흐름을 지원하는 로컬 전용 React 앱을 만든다. 앱은 Node 기반 웹 UI로 구현하고, Electron 포장을 고려해 저장 계층과 UI를 분리한다.

## 작업 순서

1. Vite React TypeScript 앱을 구성한다.
   - 검증 기준: 앱이 로컬 개발 서버에서 실행되고 프로덕션 빌드가 성공한다.

2. 도메인 타입과 순수 로직을 만든다.
   - 검증 기준: 운동 목록 항목, 루틴 스냅샷, 활성 루틴, 오늘 요일 운동 선택, 세트 완료 흐름을 테스트할 수 있다.

3. IndexedDB 저장 계층을 만든다.
   - 검증 기준: 운동 목록, 루틴, 설정, 완료 기록을 저장하고 다시 불러올 수 있다.

4. 주요 화면을 구현한다.
   - 검증 기준: 운동 진행, 운동 목록, 루틴 편집, 달력 기록, 설정 화면을 탭으로 이동할 수 있다.

5. 운동 진행 오버레이를 구현한다.
   - 검증 기준: YouTube 영역 위에 드래그 가능한 컨트롤이 있고, 세트 완료 입력이 휴식 타이머와 진행 상태를 갱신한다.

6. 테스트와 빌드를 실행한다.
   - 검증 기준: 핵심 도메인 테스트와 빌드가 통과한다.

## 첫 구현 범위

- 서버, 로그인, 동기화는 만들지 않는다.
- 세트별 장기 기록은 저장하지 않는다.
- YouTube URL은 앱에서 입력해 임베드 플레이어에 표시한다.
- 영상 임베드 실패 감지는 브라우저 제약이 있어 외부 열기 링크를 항상 제공하는 방식으로 처리한다.
- 휴식 중 새로고침 복구는 구현하지 않는다.
