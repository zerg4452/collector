# Workout Companion Design

## 목적

MacBook으로 YouTube를 보며 운동하는 사용자를 위한 로컬 전용 운동 companion을 만든다. 사용자는 기존에 MacBook으로 영상을 재생하고 휴대폰 스톱워치로 휴식 시간을 쟀다. 이 앱은 YouTube 재생, 세트 완료 입력, 휴식 타이머, 요일별 루틴 진행, 달력 완료 기록을 한 화면 흐름으로 묶는다.

첫 버전의 목표는 운동 중 입력을 줄이고, 세트 진행과 휴식 타이머를 자동화하며, 그날 운동 완료 여부를 달력에 남기는 것이다.

## 제품 방향

첫 구현은 Node 기반 웹 UI로 만든다. React와 Vite 같은 프론트엔드 구조를 우선 사용하고, Electron으로 Mac 데스크톱 앱처럼 포장할 수 있게 설계한다.

서버, 계정, 로그인, 기기 간 동기화는 만들지 않는다. 데이터는 사용자의 MacBook에만 저장한다.

## 핵심 사용 흐름

1. 앱이 활성 루틴을 확인한다.
2. 오늘 요일에 해당하는 운동 목록을 불러온다.
3. 사용자가 앱 안에서 YouTube를 재생한다.
4. 사용자가 운동 세트를 수행한다.
5. 세트가 끝나면 화면 버튼 또는 단축키로 세트 완료를 입력한다.
6. 앱이 세트 완료를 기록하고 휴식 타이머를 시작한다.
7. 휴식 시간이 끝나면 소리와 화면 강조로 알린다.
8. 사용자는 다음 세트를 직접 수행한다.
9. 다음 세트가 끝나면 다시 세트 완료를 입력한다.
10. 모든 운동의 모든 세트가 끝나면 오늘 날짜를 완료로 저장한다.

다음 세트 시작 버튼은 별도로 만들지 않는다. 앱은 세트 완료 입력과 휴식 타이머 반복에 집중한다.

## 주요 화면

### 운동 진행 화면

YouTube 재생 영역을 중심에 둔다. YouTube는 루틴과 강하게 연결하지 않고, 앱 안에서 볼 수 있는 영상 영역으로 제공한다.

YouTube 위에는 드래그 가능한 중간 크기 플로팅 컨트롤을 올린다. 컨트롤에는 현재 운동명, 현재 세트와 총 세트, 휴식 타이머, 세트 완료 버튼을 표시한다.

화면 버튼 클릭이 기본 조작 방식이다. 단축키는 보조 입력으로 제공한다.

### 운동 목록 화면

운동 목록은 운동 실행 단위를 관리한다. 항목 하나는 운동 이름, 무게, 목표 횟수, 세트 수, 휴식 시간을 가진다.

같은 운동 이름이라도 무게나 횟수가 다르면 별도 항목으로 존재할 수 있다. 예를 들어 벤치프레스 60kg 5회와 벤치프레스 50kg 3회는 서로 다른 운동 목록 항목이다.

### 루틴 편집 화면

루틴은 하나의 이름을 가진 주간 구성이다. 각 루틴은 월요일부터 일요일까지 요일별 운동 배치를 가진다.

운동 목록 항목을 요일에 배치하면, 그 항목은 루틴 안에 복사되어 고정된다. 이후 원본 운동 목록 항목을 수정해도 이미 루틴에 들어간 항목은 바뀌지 않는다.

루틴은 여러 개 만들 수 있다. 실제 운동 진행에 사용되는 활성 루틴은 한 개만 가능하다.

### 달력 기록 화면

날짜별 운동 완료 여부만 표시한다. 사용자는 운동을 했는지 안 했는지를 달력 형태로 확인한다.

첫 버전에서는 세트별 장기 기록, 중량 변화 추적, 수행 횟수 변화 추적을 만들지 않는다.

## 데이터 모델

### ExerciseItem

운동 목록의 원본 항목이다.

- `id`
- `name`
- `weight`
- `targetReps`
- `sets`
- `restSeconds`
- `createdAt`
- `updatedAt`

### RoutinePreset

완성된 주간 루틴이다.

- `id`
- `name`
- `isActive`
- `days`
- `createdAt`
- `updatedAt`

`days`는 월요일부터 일요일까지의 요일 키를 가지고, 각 요일 값은 `RoutineExerciseSnapshot` 배열이다.

### RoutineExerciseSnapshot

운동 목록 항목을 루틴에 배치할 때 만들어지는 복사본이다.

- `id`
- `sourceExerciseId`
- `name`
- `weight`
- `targetReps`
- `sets`
- `restSeconds`
- `order`

`sourceExerciseId`는 원본 추적용이다. 원본 수정이 스냅샷에 자동 반영되지는 않는다.

### WorkoutCompletion

달력 표시용 완료 기록이다.

- `date`
- `completed`
- `routinePresetId`
- `completedAt`

첫 버전에서는 세트별 상세 기록을 장기 저장하지 않는다.

### AppSettings

앱 동작 설정이다.

- `alarmVolume`
- `restEndSoundEnabled`
- `restEndVisualAlertEnabled`
- `floatingControlPosition`
- `keyboardShortcutEnabled`

## 상태 흐름

운동 진행 상태는 현재 운동 인덱스, 현재 세트 번호, 휴식 중 여부, 남은 휴식 시간, 완료 여부를 가진다.

세트 완료 입력이 들어오면 현재 세트 번호를 증가시키고 해당 운동의 휴식 시간을 기준으로 타이머를 시작한다. 휴식이 끝나면 화면 강조와 알림음을 실행하고, 다음 세트를 수행할 수 있는 상태로 돌아간다.

마지막 운동의 마지막 세트가 완료되면 오늘 날짜의 `WorkoutCompletion`을 완료로 저장한다.

## 저장 방식

`alarmVolume`은 0부터 100까지의 숫자로 저장한다.

첫 버전은 IndexedDB를 사용해 로컬 저장한다. 저장 계층은 앱 UI와 분리해서, Electron 포장 단계에서 로컬 파일이나 SQLite로 옮길 수 있게 한다.

저장 대상은 운동 목록, 루틴, 활성 루틴 정보, 앱 설정, 달력 완료 기록이다.

## 예외 처리

활성 루틴이 없으면 운동 진행 화면에서 루틴 생성 또는 활성화를 안내한다.

오늘 요일에 운동이 없으면 오늘 루틴 없음 상태를 보여준다.

YouTube 영상이 앱 안에서 재생되지 않으면 외부 브라우저로 여는 대체 동작을 제공한다.

휴식 중 앱을 새로고침하거나 재시작했을 때 진행 중 타이머 복구는 첫 버전에서 만들지 않는다. 사용자는 오늘 운동을 다시 시작한다.

운동 중 실수로 세트 완료를 눌렀을 때는 직전 동작을 되돌리는 undo를 하나 제공한다.

## 테스트 기준

운동 목록 항목을 생성, 수정, 삭제할 수 있어야 한다.

운동 목록 항목을 루틴에 배치하면 스냅샷으로 복사되어야 한다.

원본 운동 목록 항목을 수정해도 기존 루틴 스냅샷은 바뀌지 않아야 한다.

활성 루틴은 한 개만 존재해야 한다.

오늘 요일에 맞는 운동 목록을 활성 루틴에서 불러와야 한다.

세트 완료 입력이 현재 세트 상태를 올리고 휴식 타이머를 시작해야 한다.

휴식 타이머 종료 시 알림음과 화면 강조가 동작해야 한다.

알람 볼륨 설정이 저장되고 다음 실행에도 유지되어야 한다.

모든 세트 완료 시 오늘 날짜가 완료로 저장되어야 한다.

완료 기록은 달력에서 했다 또는 안 했다로 표시되어야 한다.

## 첫 버전에서 제외하는 범위

계정, 로그인, 서버 저장, 클라우드 동기화는 제외한다.

iOS 앱, Android 앱, 다른 기기 연동은 제외한다.

중량 변화 분석, 세트별 장기 기록, 실제 수행 횟수 기록은 제외한다.

YouTube 영상과 루틴의 자동 연결은 제외한다.

휴식 중 재시작 복구는 제외한다.
